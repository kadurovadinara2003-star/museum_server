**Установка необходимых библиотек**
pip install -r requirements.txt

**Запуск сервера**
uvicorn main:app --reload

Описание файлов проекта «Виртуальный экскурсовод музея»

1. main.py
Главный файл сервера на FastAPI. Содержит эндпоинт /ask, который принимает POST-запрос с id экспоната и текстом вопроса. Загружает экспонат из базы, определяет тип вопроса (через классификатор), выбирает максимальное количество токенов в зависимости от типа, строит промпт и вызывает LLM. Возвращает JSON с ответом, названием экспоната и определённым типом вопроса.

2. llm_api.py
Отвечает за взаимодействие с OpenRouter API. Содержит функцию ask_llm(prompt, max_tokens), которая отправляет запрос к выбранной модели (openai/gpt-oss-120b:free), передаёт заголовки авторизации и возвращает сгенерированный ответ. При ошибке возвращает сообщение об ошибке.

3. prompt_builder.py
Строит промпт для LLM в зависимости от типа вопроса. Принимает данные экспоната, текст пользователя и тип вопроса (short_info, full_info, history, author, fact). Добавляет базовые правила (использовать только информацию из описания, не использовать списки и таблицы, следить за длиной) и конкретные инструкции для каждого типа (ограничения по символам, акцент на истории, авторе или факте). Возвращает готовую строку промпта.

4. question_classifier.py
Эвристический классификатор типов вопросов. Функция classify_question(text) анализирует текст вопроса: проверяет ключевые слова для типов author, history, fact, затем для full_info, и наконец для short_info (короткий вопрос или наличие слов «где», «когда», «сколько»). Возвращает строку с типом. Если ни одно правило не сработало, возвращает full_info.

5. database.py
Загружает данные экспонатов из файла data/exhibits.json при старте сервера. Содержит функцию get_exhibit_by_id(exhibit_id), которая ищет экспонат по id в загруженном списке и возвращает его словарь или None.

6. collect_answers.py
Скрипт для массового сбора ответов системы. Читает тестовые вопросы из test_questions.json, отправляет каждый вопрос на локальный сервер (http://127.0.0.1:8000/ask), сохраняет ответы в CSV-файл на рабочий стол (answers.csv) и открывает папку с файлом. Использует таймаут 120 секунд и паузу 1 секунду между запросами.

7. evaluate_classifier.py
Скрипт оценки точности классификатора. Загружает test_questions.json, для каждого вопроса вызывает classify_question и сравнивает полученный тип с ожидаемым. Выводит общее количество примеров, правильные ответы, Accuracy, а для каждого типа – Precision, Recall, F1-меру и количество реальных/предсказанных примеров.

8. requirements.txt
Список зависимостей Python для запуска сервера: fastapi, uvicorn, requests, python-dotenv. (При необходимости можно добавить pandas и openpyxl для работы со скриптами оценки.)

9. test_questions.json
Файл с тестовым набором из 50 вопросов. Каждый вопрос содержит id_exp (id экспоната), question_text (текст вопроса), expected_type (ожидаемый тип). Используется как для оценки классификатора, так и для сбора ответов LLM.

10. llm_api.py (дополнительно)
В файле указан API-ключ, который необходимо заменить на свой (начинается с sk-or-v1-...).
